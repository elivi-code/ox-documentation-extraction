## Installation

```sh
# yarn
yarn add @8x8/oxygen-toggle-button

# npm
npm install @8x8/oxygen-toggle-button
```

**Registry prerequisite** — add to `.npmrc` (or `.yarnrc.yml` for yarn):
```
@8x8:registry=https://artifactory.es.8x8.com/artifactory/api/npm/npm-repository/
```
VPN required to access the private Artifactory registry.

|-----------|-------------|-------------|
| `ToggleButton` | default | Toggle switch with label, info tooltip, and indeterminate state |

## ToggleButton props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `testId` | `string` | `'TOGGLE_BUTTON'` | Test id applied to the input element |
| `id` | `string` | autogenerated | Value for the `id` input attribute and `htmlFor` label attribute |
| `label` | `node` | `null` | Label content — text or a custom component |
| `value` | `number \| string \| bool \| { id: string }` | `''` | Input value passed on change |
| `name` | `string` | — | Name attribute for the input |
| `infoBoxText` | `node` | — | Renders an info icon; content shown inside a Tooltip on click |
| `infoBoxButtonLabel` | `string` | — | Accessible `aria-label` for the info icon button |
| `isChecked` | `bool` | `false` | Controlled checked state |
| `isDisabled` | `bool` | `false` | Disables the toggle and applies disabled visual treatment |
| `isIndeterminate` | `bool` | `false` | Puts the toggle in an indeterminate state |
| `showLabelTooltipOnOverflow` | `bool` | `true` | Shows a tooltip when the label text overflows its container |
| `onBlur` | `func` | `noop` | Blur synthetic event handler |
| `onChange` | `func` | `noop` | Called when the toggle value changes; receives the new value |
| `onFocus` | `func` | `noop` | Focus synthetic event handler |

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `isHorizontal` | `boolean` | No | — | Lays toggle items out horizontally instead of vertically |
| `children` | `ReactNode` | No | — | `ToggleButton` children |

---

## Notes

- `ToggleButton` is a **controlled component** — manage `isChecked` state in the parent and update it via `onChange`.
- `id` is autogenerated if not supplied; supply it explicitly when you need a stable `htmlFor` / `aria-labelledby` pairing.
- `infoBoxText` + `infoBoxButtonLabel` must be supplied together — the button label is required for accessibility when the info icon is shown.
