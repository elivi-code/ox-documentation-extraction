Skeleton Circle — Usage Guidelines

See also: props.md · examples.md · tokens.md · accessibility.md · SkeletonCircle-figma.md · SkeletonBlock-usage.md

Editorial guidance — not Figma-sourced. No ↳ Skeleton loader examples Figma page exists, so figma-extract-usage cannot run. The Do/Don't pairs below are inferred from props.md, examples.md, accessibility.md, tokens.md, SkeletonCircle-figma.md, and the public usage page at oxygen.8x8.com/components/skeletonloader/usage. Treat as provisional until a designer reviews them. Replace with figma-extract-usage output when a Figma examples page is created.

SkeletonCircle is a fully-rounded content placeholder that appears while an avatar or individual icon is loading. It is one half of the Oxygen skeleton family — pair it with SkeletonBlock for text-line and image placeholders in the same layout. Both components fade between the ui01 and ui02 tokens to convey activity until the real content paints.

✅ When to use

  • Replacing an avatar (xlarge / large / medium / small — 80px → 28px) while a user, contact, or profile loads.
  • Replacing an individual icon in a small slot (24px → 16px) — toolbar, list-item leading icon, status indicator.
  • The loaded element is round in the design — circle skeletons must visually match circular avatars, not square thumbnails.
  • The page or list is incrementally loading and a circular placeholder keeps the row layout stable.

❌ When not to use

  • Element is an image, thumbnail, or hero → use SkeletonBlock size="custom - image"
  • Element is a text line → use SkeletonBlock with the text-style size value
  • Container content cannot be determined → use Spinner
  • Loading inside a menu, dropdown, or modal → use Spinner
  • You have a real percentage (0–100%) → use Progress Bar
Per oxygen.8x8.com/components/skeletonloader/usage: "Circle skeleton loaders are used for avatars and individual icons."

Do / Don't

Pair 1 — Match the diameter to the slot
✅ Do — Pick the size value whose diameter matches the loaded avatar / icon

SkeletonCircle ships nine diameter values: 80px - xlarge, 48px - large, 40px - medium, 32px - small, 28px - small, and four xsmall variants at 24/22/20/16px. Use the one matching the avatar tier being replaced, so the row keeps its height when the real avatar paints.

<SkeletonCircle size="80px - xlarge" />  {/* header */}
<SkeletonCircle size="48px - large" />   {/* list */}
<SkeletonCircle size="40px - medium" />  {/* compact */}

Grounded in: props.md size table; SkeletonCircle-figma.md — "Each size value encodes both the pixel dimension and the semantic size label"; oxygen.8x8.com — 9 Circle sizes from 80px to 16px.

❌ Don't — Default every avatar to the same diameter regardless of slot

Rendering a 40px medium circle in a slot that will load an 80px header avatar means the page reflows when the real avatar paints — the header row visibly grows.

Pair 2 — Circle is for avatars and icons
✅ Do — Reserve SkeletonCircle for avatars and individual icons

The Oxygen Skeleton family has two shape primitives because real UI has two shape primitives: round avatars/icons (Circle) and rectangular text/images (Block). Use them the same way — never as interchangeable.

{/* Avatar */}
<SkeletonCircle size="48px - large" />

{/* Square thumbnail — use Block instead */}
<SkeletonBlock size="custom - image" />

Grounded in: oxygen.8x8.com — Circle: "used for avatars and individual icons"; Block: "typically used for images"; SkeletonCircle-figma.md — shared ui02 token but distinct shape primitives.

❌ Don't — Use a large SkeletonCircle to stand in for a square image or thumbnail

Showing a circle where a square will paint misrepresents the shape of the loaded UI. The layout transforms from round to square when the real content arrives — visually jarring.

Pair 3 — One diameter per collection
✅ Do — Use a single, consistent diameter across every row of a list or grid

A contact list or chat history renders all avatars at the same size. The skeleton should mirror that — every row uses the same SkeletonCircle size so the loading state reads as a single rhythmic column.

{rows.map(r => (
  <div key={r.id} style={{ display: 'flex',
        alignItems: 'center', gap: '12px' }}>
    <SkeletonCircle size="40px - medium" />
    <SkeletonBlock size="20px - body01" />
  </div>
))}

Grounded in: examples.md contact-row example; SkeletonCircle-figma.md — "pixel values are the disambiguating factor"; oxygen.8x8.com — "On large container-based components like tables or cards."

❌ Don't — Mix sizes within one collection

Rendering some rows with 48px - large and others with 28px - small produces a loading state that doesn't predict the loaded state. When the real avatars paint at one consistent size, every row reflows independently.

Pair 4 — Pair Circle with Block lines
✅ Do — Combine SkeletonCircle with SkeletonBlock lines for contact rows and list items

SkeletonCircle rarely stands alone — it represents the avatar slot in a row that also has a name and a secondary line. Wrap one circle and two blocks in a flex row to skeleton the entire pattern.

<div style={{ display: 'flex',
              alignItems: 'center', gap: '12px' }}>
  <SkeletonCircle size="40px - medium" />
  <div style={{ flex: 1, display: 'flex',
                flexDirection: 'column', gap: '8px' }}>
    <SkeletonBlock size="20px - body01" />
    <SkeletonBlock size="16px - label01" />
  </div>
</div>

Grounded in: examples.md "Combined with SkeletonBlock (contact row)"; SkeletonBlock/examples.md mirror Card-skeleton example; oxygen.8x8.com — "The spacing tokens used in the 'loaded' state will be used as default to separate different variants."

❌ Don't — Use a lone SkeletonCircle where the loaded UI has a circle and text

A row that loads as "avatar + name + status" but is skeletoned as a single circle leaves a large blank space to the right. The skeleton fails to preserve the row's layout, so the page jumps on load.

Pair 5 — Announce loading on the parent
✅ Do — Set aria-busy="true" on the parent container, once per loading region

SkeletonCircle is purely presentational — a styled div with no role, no text, no tab stop. Communicate loading on the parent that wraps the row (or the whole list) — once, not per skeleton.

<ul aria-busy="true" aria-label="Loading contacts">
  <li>
    <SkeletonCircle size="40px - medium" />
    <SkeletonBlock size="20px - body01" />
  </li>
</ul>

Grounded in: accessibility.md — ARIA role section, Screen reader guidance, WCAG 4.1.3 Status Messages.

❌ Don't — Add aria-label="Loading" or role="status" to every individual circle

Putting a label on every skeleton makes screen readers read "Loading. Loading. Loading. Loading…" across a 20-row list — drowning out the surrounding content. One label on the parent is correct; per-skeleton labels are noise.

Pair 6 — Honour prefers-reduced-motion
✅ Do — Verify the shimmer pauses when the user has requested reduced motion

The skeleton animation fades the circle's fill between ui01 and ui02. Looping animations can trigger vestibular discomfort, so prefers-reduced-motion: reduce exists. Confirm the circle renders as a static ui02 disc when the preference is set.

Grounded in: accessibility.md — Animation considerations; WCAG 2.3.3; oxygen.8x8.com — "alternating the colors from ui01 to ui02."

❌ Don't — Re-apply a custom shimmer in a parent stylesheet without gating on motion preferences

Wrapping the skeleton in a parent that re-applies its own @keyframes overrides the Oxygen reduced-motion behaviour. Any custom shimmer must itself be wrapped in @media (prefers-reduced-motion: no-preference).
