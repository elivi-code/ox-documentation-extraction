Badge is a passive display component for notification counts and presence indicators. It renders as a small pill or dot — it is never interactive and never focusable. Two sub-components exist: Badge for numeric/dot indicators and AIBadge for AI-feature labelling.

Do / Don't

Pair 1 — Counter badge content
✅ Do — Pass only numeric values as children

The children prop is documented as "numerical value only". Use whole numbers for up to two digits; use "99+" for counts that exceed two digits.

<Badge>4</Badge> <Badge>12</Badge> <Badge>99+</Badge>

Grounded in: props.md — children description: "Rendered element inside the badge (numerical value only)"

❌ Don't — Put non-numeric content in a counter badge

Avoid icons, arbitrary strings, or emoji as children. The counter badge is sized and styled for numeric values; other content overflows or renders incorrectly.

{/* Wrong */} <Badge>New</Badge> <Badge>★</Badge>
Pair 2 — Dot badge
✅ Do — Omit children and use size="small" for a dot indicator

A dot badge signals presence or activity without a numeric count. Render it with no children and size="small".

<Badge size="small" />

Grounded in: examples.md — "Use a badge without children and size="small" to render a dot indicator."

❌ Don't — Add children to a dot badge

Passing text or numbers to a size="small" badge turns it into a counter and breaks the dot affordance — the badge grows to accommodate the label and loses its minimal dot form.

{/* Wrong — renders counter, not dot */} <Badge size="small">1</Badge>
Pair 3 — AI feature labelling
✅ Do — Use AIBadge for AI-related features

AIBadge renders a star icon alongside text and is purpose-built for AI feature labelling. It accepts custom label text and inherits all standard HTML div attributes.

import { AIBadge } from '@8x8/oxygen-badge'; <AIBadge /> {/* "AI" with star */} <AIBadge>Beta</AIBadge>

Grounded in: props.md — AIBadge is "a specialised badge for AI-related features… displays a star icon alongside text content."

❌ Don't — Use Badge to label AI features

Badge is for notification counts and presence indicators. Using it for AI labels loses the semantic star icon, misrepresents the component's purpose, and diverges from the established pattern.

{/* Wrong */} <Badge>AI</Badge>
Pair 4 — Conditional visibility
✅ Do — Render badge only when there is something to signal

Badges should appear only when relevant — when there are unread items, an active meeting, or a notification count above zero. Conditional rendering keeps the DOM clean and avoids false positive cues.

{hasUnread && ( <Badge size="small" aria-hidden="true" /> )} {unreadCount > 0 && ( <Badge>{unreadCount}</Badge> )}

Grounded in: examples.md — "Badges are not permanently rendered — show them based on application state."

❌ Don't — Keep an empty badge permanently in the DOM

Rendering <Badge></Badge> or an empty dot at all times produces visual noise for sighted users and may produce spurious announcements for screen reader users depending on live region configuration.

{/* Wrong — always renders, even with no count */} <Badge>{unreadCount}</Badge>
Pair 5 — Accessibility
✅ Do — Hide the badge and carry the count on the parent

Badge is a passive visual element. Apply aria-hidden="true" to the badge itself and embed the count in the parent's aria-label. For dynamically updated counts, add aria-live="polite" to the parent region.

<IconButton icon={<BellIcon />} aria-label={`Notifications, ${count} unread`} > <Badge isInner aria-hidden="true"> {count} </Badge> </IconButton>

Grounded in: accessibility.md — badge is non-interactive, aria-hidden; parent carries accessible name.

❌ Don't — Rely on the badge text as the only accessible count

If the badge is the only place the count appears in the accessibility tree, screen reader users navigating by landmark or button will hear the parent label without any count, then encounter a bare number badge out of context.

{/* Wrong — count only in badge, not on parent */} <IconButton icon={<BellIcon />} aria-label="Notifications"> <Badge isInner>{count}</Badge> </IconButton>
