Spinner — Usage Guidelines

See also: props.md · examples.md · tokens.md · accessibility.md · Spinner-figma.md · Spinner-audit.md

Editorial guidance — not Figma-sourced. No ↳ Spinner examples Figma page exists, so figma-extract-usage cannot run. The Do/Don't pairs below are drafted from the public Oxygen usage page at oxygen.8x8.com/components/spinner/usage together with props.md, examples.md, and accessibility.md. Treat as provisional until a designer reviews them. Replace with figma-extract-usage output when a Figma examples page is created.

Spinner is a continuously rotating arc that signals an indeterminate background task — "loading sections or areas such as a page or a dropdown menu." It is passive and non-interactive; its only job is to tell the user wait, something is happening. The animation stops once loading completes.

Anatomy

  1. Spinner arc — the rotating circular indicator. Renders at a named size ("small", "default", "large2x") or a raw pixel number via the size prop. Stroke width is adjustable via strokeWidth.
  2. Supporting text (strongly recommended) — a short label rendered next to the arc, passed via the text prop. The Oxygen usage page is explicit: "There should always be supporting text where space permits, as it is best practice to give clear expectations to the user." When text is provided it acts as the accessible label — no separate aria-label is needed.

✅ When to use

  • A whole page or large section is loading and the structure is unknown.
  • Modal content is being fetched before the modal can populate.
  • A menu, dropdown, or overflow menu is hydrating from a network call — use size="small".
  • A card or row is loading in place — inline, compact contexts.

❌ When not to use

  • Determinate progress — if you can show a percentage, use ProgressBar instead.
  • Known content shape — prefer Skeleton loaders to prevent layout shift.
  • Very short waits (<200 ms) — a flashing spinner reads as a bug.
  • Decoration — Spinner is for loading state only.
Per oxygen.8x8.com/components/spinner/usage: "Spinners are loading circles that are used for loading sections or areas such as a page or a dropdown menu… When there is a determinate value. Use a progress bar instead."

Do / Don't

Pair 1 — Indeterminate waits vs. measurable progress
✅ Do — Use Spinner when you cannot report a percentage

Spinner is for indeterminate background work — fetching a record, bootstrapping a page, loading a dropdown. The animation signals "something is happening" when you cannot tell the user how long is left.

{/* Right — fetching a record; duration unknown */}
{isLoading
  ? <Spinner size="default" text="Loading record" />
  : <RecordView record={record} />}

Grounded in: oxygen.8x8.com — "Spinners are loading circles… used for loading sections or areas."

❌ Don't — Use Spinner where you have a real percentage

A spinner over a file upload tells the user nothing. If you can compute how much work is done, show it — use a ProgressBar (or equivalent) so the user knows how long is left.

{/* Wrong — measurable progress shown as a spinner */}
<Spinner size="default"
  text={`Uploading… ${bytesSent}/${totalBytes}`} />

{/* Right — use ProgressBar for known-percentage work */}
<ProgressBar value={bytesSent} max={totalBytes}
  label="Uploading" />

Grounded in: oxygen.8x8.com — "When there is a determinate value. Use a progress bar instead."

Pair 2 — Match size to the area being loaded
✅ Do — Use size="small" for inline contexts, size="default" for page/section loads

The Oxygen usage page defines two clear roles: default "used typically when a whole page is being loaded" and small "may also be used for inline loading when only a small part of the UI is being loaded (e.g. cards or overflow menus)." Pick the size from the area being loaded.

{/* Inline — dropdown is hydrating */}
<DropdownMenu>
  {options.length === 0
    ? <Spinner size="small" text="Loading options" />
    : options.map(renderOption)}
</DropdownMenu>

{/* Page / large section */}
{isLoading
  ? <Spinner size="default" text="Loading conversation" />
  : <Conversation thread={thread} />}

Grounded in: oxygen.8x8.com (Variants — default / small); props.md spinnerSize keys; examples.md size examples.

❌ Don't — Use an oversized spinner inside a small container, or a tiny spinner for a full page

A size="default" spinner inside a 32 px dropdown row eats the menu. A size="small" spinner centred on an empty full-page view looks like a bug, not a loading state.

{/* Wrong — page-sized spinner inside a dropdown row */}
<DropdownMenu>
  <Spinner size="default" />
</DropdownMenu>

{/* Wrong — tiny spinner on a full-page loading state */}
<main style={{ minHeight: '80vh' }}>
  <Spinner size="small" />
</main>

Grounded in: oxygen.8x8.com (Variants); examples.md size guidance.

⚠️ "large2x" is documented in props.md (from Storybook) but is absent from the Figma component set (audit GAP-011). Prefer "default" and "small" for new work.

Pair 3 — Always pair with supporting text where space permits
✅ Do — Use the text prop so the user knows what is loading

A label answers the user's real question — "is the system frozen, or is this specific thing loading?" The Oxygen usage page is explicit: "There should always be supporting text where space permits." When you use text, it also acts as the accessible label — no separate aria-label needed.

{/* Right — text scopes the wait to a specific thing */}
<Spinner size="default"
  text="Loading conversation history" />

{/* Right — inline contexts: visible surrounding label is enough,
     fall back to aria-label when text prop can't fit */}
<DropdownLabel>
  Contacts
  <Spinner size="small"
    aria-label="Loading contacts" />
</DropdownLabel>

Grounded in: oxygen.8x8.com (anatomy — "There should always be supporting text where space permits"); props.md text; accessibility.md.

❌ Don't — Render a bare spinner with no label of any kind

A spinner with neither text nor aria-label is completely silent to screen readers and ambiguous to sighted users. Even the fallback aria-label="Loading" is weaker — it doesn't scope the wait to anything meaningful.

{/* Wrong — no label; invisible to screen readers */}
<Spinner size="default" />

{/* Weaker — passes a11y but tells the user nothing specific */}
<Spinner size="default" aria-label="Loading" />

Grounded in: accessibility.md — "Always supply either text or aria-label. A spinner without any label is invisible to screen reader users."; WCAG 1.1.1 Non-text content.

Pair 4 — Respect prefers-reduced-motion via hasAnimation
✅ Do — Disable the rotation for users who opt out of motion

Spinner's continuous rotation is exactly the kind of animation users with prefers-reduced-motion: reduce are opting out of. Wiring hasAnimation={false} to the media query preserves the label meaning while removing the motion.

const prefersReducedMotion =
  useMediaQuery('(prefers-reduced-motion: reduce)');

<Spinner
  size="default"
  text="Loading record"
  hasAnimation={!prefersReducedMotion}
/>

Grounded in: accessibility.md — Motion sensitivity section; WCAG 2.3.3 (Animation from Interactions).

❌ Don't — Leave the rotation running for motion-sensitive users

Ignoring prefers-reduced-motion on a continuously spinning element is a WCAG 2.3.3 failure and a poor experience for users with vestibular disorders.

{/* Wrong — rotation runs regardless of OS preference */}
<Spinner size="default" text="Loading record" />

Grounded in: accessibility.md Motion sensitivity; audit WARN-002.

⚠️ It is currently unconfirmed whether @8x8/oxygen-loaders reads the OS prefers-reduced-motion preference automatically or requires the consuming app to wire hasAnimation={false} manually (audit WARN-002). Until verified against the package source, do the wiring yourself.
Pair 5 — Prefer Skeleton when the content shape is known
✅ Do — Use Spinner only when the structure of incoming content is unknown

When you know the shape of what's coming — rows, cards, a chart — Skeleton loaders reduce perceived wait, prevent layout shift, and feel more intentional. Spinner is for shapeless, unbounded waits.

{/* Right — shape unknown (page bootstrap) */}
{isBootstrapping
  ? <Spinner size="default"
      text="Loading workspace" />
  : <Workspace />}

{/* Right — shape known; prefer skeletons */}
{isLoading
  ? <ConversationListSkeleton rowCount={10} />
  : <ConversationList items={items} />}

Grounded in: oxygen.8x8.com (Related components — Skeleton loader).

❌ Don't — Drop a Spinner into a surface where a Skeleton would carry more meaning

A spinner centred over a known list shape throws away the chance to anchor the user — the page jumps when content arrives, and the user has no sense of "how many?" or "how tall?" until it lands.

{/* Wrong — known list shape, no skeleton, layout jumps */}
{isLoading
  ? <div style={{
        display: 'grid',
        placeItems: 'center',
        minHeight: 400,
      }}>
      <Spinner size="default"
        text="Loading conversations" />
    </div>
  : <ConversationList items={items} />}

Grounded in: oxygen.8x8.com (Related components — Skeleton loader); "When not to use" — content-shaped placeholders preferred.

Pair 6 — One spinner per loading region; never decorative
✅ Do — Show one spinner anchored to the active loading scope

Show exactly one spinner per loading region and scope it to the right level. If a whole page is loading, show a page-level spinner. If only a card inside that page is independently loading, show a card-level spinner instead — not both at once.

{/* Right — one spinner at a time, scoped to the active wait */}
{pageLoading
  ? <Spinner size="default" text="Loading workspace" />
  : <Workspace>
      {cardLoading
        ? <Spinner size="small" text="Loading metrics" />
        : <MetricsCard metrics={metrics} />}
    </Workspace>}

Grounded in: oxygen.8x8.com — "a seamless and continuous animation that will stop once the page is loaded."

❌ Don't — Stack spinners or use Spinner as decoration

Multiple competing spinners on one screen create noise without telling the user what they're waiting on. A spinner used purely for visual interest (e.g. as an illustration in an empty state) misrepresents that loading is in progress.

{/* Wrong — three concurrent spinners; unclear which to wait for */}
<Spinner size="default" text="Loading" />
<Spinner size="small" text="Loading metrics" />
<Spinner size="small" text="Loading recents" />

{/* Wrong — decorative; nothing is loading */}
<EmptyState
  illustration={<Spinner size="large2x" />}
  title="No conversations yet"
/>

Grounded in: oxygen.8x8.com — Behavior; "When not to use" — Spinner is for loading state only.
